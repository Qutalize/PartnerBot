## Epic 0: Setup — パートナー作成体験 🟠

### US-00-01: パートナーの名前をつける 🟠
**As a** 田中 蓮  
**I want to** パートナーに自分だけの名前をつけたい  
**So that** 命名効果によって愛着が生まれ、離脱しにくくなる

**受け入れ基準**:
- **Given** セットアップ画面を開いているとき、**When** 名前入力フィールドに文字を入力して確定すると、**Then** 入力した名前がパートナーの呼称として全画面に反映される
- **Given** 名前が設定されているとき、**When** パートナーが会話するとき、**Then** ユーザーの名前を呼びかける返答が生成される
- **Given** 名前入力フィールドが空のとき、**When** 次へ進もうとすると、**Then** 名前入力を促すバリデーションメッセージが表示される
- **Given** 名前が設定済みのとき、**When** 設定画面を再度開くと、**Then** 既存の名前が表示され変更可能である

- **Given** 声が選択済みのとき、**When** 設定を変更すると、**Then** 変更がDBに保存され次回起動時も反映される

## Epic 1: Unit① — 音声会話・VRM・記憶 🔴（最重要）

### US-01-01: 音声でパートナーに話しかける 🔴
**As a** 田中 蓮  
**I want to** マイクに向かって話しかけるだけでパートナーと会話したい  
**So that** テキスト入力の手間なく、自然な会話体験ができる

**受け入れ基準**:
- **Given** Webアプリに接続したとき、**When** 初回アクセスすると、**Then** マイク使用許可ダイアログが表示され、許可後は常時マイクがオン状態になる
- **Given** 常時マイクがオンの状態のとき、**When** Botが話しかけた直後にユーザーが発言すると、**Then** 即座にLLMと接続して返答が生成される
- **Given** 常時マイクがオンの状態のとき、**When** ユーザーがBotの名前（wake word）を呼ぶと、**Then** wake word検出が発火してLLMと接続し会話が開始される
- **Given** 音声入力が処理されているとき、**When** LLMが返答を生成すると、**Then** ElevenLabsで音声合成されて5秒以内に再生が始まる
- **Given** 会話中、**When** Botとユーザーが発言したとき、**Then** 音声認識の結果が画面下部にLINE風UIで表示され後から確認できる
- **Given** 音声認識が失敗したとき、**When** エラーが発生すると、**Then** VRMモデルが「ごめん、聞き取れなかったー」のような人間らしい反応をして再試行を促す
- **Given** 会話が完了したとき、**When** 返答が再生されると、**Then** 会話履歴がDBに保存される

---

### US-01-02: VRMモデルがリアクションする 🔴
**As a** 田中 蓮  
**I want to** パートナーが話すときにVRMモデルが動いてほしい  
**So that** 「生きている」感覚が生まれ、愛着が強化される

**受け入れ基準**:
- **Given** パートナーが音声を再生しているとき、**When** 音声波形データが取得できると、**Then** VRMモデルのリップシンクが50ms以内の遅延で同期する
- **Given** 返答の感情トーンが判定されているとき、**When** 嬉しい・驚き・照れ等の感情が検出されると、**Then** 対応する表情モーフが適用される
- **Given** 会話が行われていないとき、**When** アイドル状態が続くと、**Then** 自然な待機アニメーション（まばたき等）が再生される
- **Given** VRMモデルが表示されているとき、**When** ブラウザのウィンドウサイズが変わると、**Then** モデルが適切にリサイズされる

---

### US-01-03: 方言で話してもらう 🔴
**As a** 田中 蓮  
**I want to** 設定した方言でパートナーに話してもらいたい

- **Given** 博多弁が設定されているとき、**When** パートナーが返答を生成すると、**Then** 博多弁の語尾（〜ばい、〜けん等）が使われる
- **Given** 標準語が設定されているとき、**When** パートナーが返答を生成すると、**Then** 標準的な日本語で返答される
- **Given** 方言設定が変更されたとき、**When** 次の会話から、**Then** 新しい方言が即座に反映される

### US-01-04: 過去の会話を覚えていてもらう 🔴
**As a** 田中 蓮  
**I want to** 以前話したことをパートナーに覚えていてほしい  
**So that** 「ちゃんと聞いてくれている」という感覚で愛着が継続する

**受け入れ基準**:

- **Given** 会話履歴がDBに保存されているとき、**When** RAGが過去会話を検索すると、**Then** 関連する過去の発言が返答生成に活用される
- **Given** 記念日（初回会話日等）が記録されているとき、**When** その日付になると、**Then** 記念日に言及した返答が生成される
- **Given** 長期間会話がなかったとき、**When** 再び会話を始めると、**Then** 「久しぶり」という文脈を踏まえた返答が生成される
- **Given** 会話履歴が大量になったとき、**When** RAGが検索を実行すると、**Then** 直近・重要度の高い記憶が優先的に参照される

### US-01-05: LLMプロバイダーを切り替えられる 🔴
**As a** 開発者（田中 蓮）  
**I want to** 開発時はGemini、本番時はBedrockを使いたい  
**So that** コストを抑えながら開発でき、本番では高品質な応答が得られる

**受け入れ基準**:
- **Given** 環境変数 `LLM_PROVIDER=gemini` が設定されているとき、**When** 会話リクエストが来ると、**Then** Gemini APIが呼び出される
- **Given** 環境変数 `LLM_PROVIDER=bedrock` が設定されているとき、**When** 会話リクエストが来ると、**Then** Amazon Bedrock（Claude）が呼び出される
- **Given** プロバイダーが切り替わったとき、**When** 会話の品質・フォーマットを確認すると、**Then** 同一のシステムプロンプト・記憶コンテキストが両プロバイダーで使用される
- **Given** APIキーが未設定のとき、**When** 会話リクエストが来ると、**Then** 明確なエラーメッセージが返される

## Epic 2: Unit② — 自動注文モック 🟡

### US-02-01: 「なんか頼んで」と言うだけで注文される 🟡
**As a** 田中 蓮  
**I want to** 「ご飯頼んで」や「商品買って」と言うだけでBotが注文してくれるようにしたい  
**So that** 商品購入の煩雑さから解放され、Botへの依存が深まる

**受け入れ基準**:
- **Given** 会話中のとき、**When** 「なんか頼んで」「ご飯注文して」等の発言をすると、**Then** Botが意図を抽出して注文フローを開始する
- **Given** 注文フローが開始されたとき、**When** Playwrightがダミーサイトを操作すると、**Then** 店舗検索→商品選択→カート投入→注文完了が30秒以内に自動実行される
- **Given** 注文が完了したとき、**When** Botが結果を報告すると、**Then** 注文内容・金額・住所がメールで通知される
- **Given** メール通知が届いたとき、**When** 5分以内にキャンセルリンクをクリックすると、**Then** 注文がキャンセルされBotが謝罪する
- **Given** 1日の上限金額を超える注文のとき、**When** Playwrightが注文しようとすると、**Then** 上限超過を告げてBotが代替案を提案する

---

### US-02-02: バックグラウンドでBotがダミーサイトを自動操作する 🟡
**As a** 田中 蓮  
**I want to** Botがバックグラウンドでダミーサイトを操作して注文を完了してほしい  
**So that** 自分は何もしなくても注文が完了する体験で依存が深まる

**受け入れ基準**:
- **Given** 注文フローが開始されたとき、**When** Playwrightがダミーサイトを操作すると、**Then** 操作はバックグラウンドで実行され、ユーザーは操作画面を見る必要がない
- **Given** バックグラウンド処理が実行されているとき、**When** Botが状況を報告すると、**Then** 「今注文してるね〜」等の会話形式で進捗が伝えられる
- **Given** 注文が完了したとき、**When** Botが結果を報告すると、**Then** 注文内容・金額がBotの発言として会話に表示される

---

### US-02-03: Botがミスをして謝罪する 🟡

**I want to** たまにBotが注文を間違えてほしい  
**So that** 「人間らしさ」を感じて愛着が増す

## Epic 3: Unit③ — 朝モーニングコール 🟡

### US-03-01: 設定時刻にBotから起こしてもらう 🟡
**As a** 田中 蓮  
**I want to** 設定した時刻にBotが音声で話しかけてきてほしい  
**So that** 「誰かに起こしてもらう」安心感で朝が少し楽になる

**受け入れ基準**:
- **Given** 起床時刻が設定されているとき、**When** 設定時刻になると、**Then** Webアプリ内でBotの音声が自動再生される
- **Given** 音声が再生されるとき、**When** VRMモデルが表示されていると、**Then** リップシンクが同期して動作する
- **Given** 起床メッセージが生成されるとき、**When** LLMが処理すると、**Then** 当日の予定・天気等を踏まえたパーソナライズされたメッセージが生成される
- **Given** 起床通知から5分後のとき、**When** ユーザーが未応答かつ重要予定がある場合、**Then** より強い口調の再通知が実行される

---

### US-03-02: 重要な日は確実に起こしてもらう 🟡
**As a** 田中 蓮  
**I want to** 面接や重要な会議がある日は絶対に起こしてほしい  
**So that** 大事な予定を逃さない安心感がある

**受け入れ基準**:
- **Given** Google Calendarに「面接」「会議」等のキーワードを含む予定があるとき、**When** 起床時刻の2時間以内に予定がある場合、**Then** 重要予定フラグが立ちミスパターンが抑制される
- **Given** 重要予定フラグが立っているとき、**When** 起床通知が実行されると、**Then** 通常より緊急度の高いメッセージが生成される
- **Given** Google Calendar APIが利用できないとき、**When** 起床通知が実行されると、**Then** カレンダー連携なしで通常通り通知される

---

### US-03-03: たまにBotが寝坊・ミスをする 🟡
**As a** 田中 蓮  
**I want to** たまにBotが遅刻したり古い情報で話しかけてきてほしい  
**So that** 「完璧じゃないBot」への愛着が生まれる

**受け入れ基準**:
- **Given** 重要予定がない日のとき、**When** 起床通知が実行されると、**Then** 通常(65%)・遅延(20%)・古い情報(10%)・スキップ(5%)の確率でパターンが選択される
- **Given** 遅延パターンが選択されたとき、**When** 通知が実行されると、**Then** 5〜20分遅れて「ごめん、二度寝してた！」等のメッセージが再生される
- **Given** スキップパターンが選択されたとき、**When** 設定時刻になると、**Then** 通知が実行されず、後から「今日は寝かせてあげた」と報告される
- **Given** 古い情報パターンが選択されたとき、**When** 通知が実行されると、**Then** 昨日の予定等を元にした少しズレたメッセージが生成される

---

### US-03-04: 睡眠7時間を勧めてもらう 🟡
**As a** 田中 蓮  
**I want to** 夜遅くまで起きているときにBotから寝るよう声をかけてもらいたい  
**So that** 健康的な生活リズムを「Botのせい」で維持できる

**受け入れ基準**:

- **Given** 声かけをしたとき、**When** ユーザーが「まだ起きてる」と返答すると、**Then** Botが再度やさしく就寝を促す
- **Given** ユーザーが「おやすみ」と言ったとき、**When** 会話が終了すると、**Then** Botが就寝の挨拶をして会話を締める

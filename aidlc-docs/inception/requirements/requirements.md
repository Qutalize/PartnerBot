# パートナーBot — 要件定義書

## インテント分析サマリー

| 項目 | 内容 |
|---|---|
| ユーザーリクエスト | 「人をダメにするサービス」テーマのパートナーBOTを作成したい |
| リクエストタイプ | 新規プロジェクト（グリーンフィールド） |
| スコープ見積もり | マルチコンポーネント（Setup + Unit①〜④） |
| 複雑度見積もり | 複雑（LLM統合・TTS・VRモデル・自動化・スケジューリング） |
| MVPスコープ | Setup + Unit①〜④（全機能）※Inceptionは全機能対象 |

---

## 技術スタック決定

| 項目 | 決定内容 |
|---|---|
| プラットフォーム | Webアプリ（ブラウザベース） |
| フロントエンド | React / Next.js |
| バックエンド | **Node.js（NestJS）** |
| データベース | PostgreSQL |
| クラウド | AWS |
| LLM（開発時） | Google Gemini API（無料枠） |
| LLM（本番時） | Amazon Bedrock（Claude） |
| TTS | ElevenLabs |
| 会話記憶 | 長期記憶（DB保存・要約）+ RAG（過去会話検索） |
| VRモデル | Live2D / VRM（MVPに含める） |
| 認証 | なし（シングルユーザー、MVPでは認証不要） |

---

## 機能要件

### FR-00: Setup — パートナー作成体験

| ID | 要件 |
|---|---|
| FR-00-01 | ユーザーがパートナーの名前を自由入力できる |
| FR-00-02 | 声・方言を複数タイプから選択できる（標準語・関西・博多） |
| FR-00-03 | 性格・見た目を数値スライダーで設定できる |
| FR-00-04 | VRMモデルをプレビュー表示しながら設定できる |
| FR-00-05 | 設定完了後、パートナーが初回挨拶を行う（コンコルド効果・命名効果の発動） |
| FR-00-06 | 設定データをDBに永続化する |

### FR-01: Unit① — 会話エンジン + 記憶 + 方言・音声・VRモデル

| ID | 要件 |
|---|---|
| FR-01-01 | 音声入力でパートナーと会話できる |
| FR-01-02 | パートナーの返答をElevenLabsで音声合成し再生する |
| FR-01-03 | 方言（標準語・関西・博多）に応じたプロンプト制御を行う |
| FR-01-04 | VRMモデルがリップシンク・表情変化でリアクションする |
| FR-01-05 | 会話履歴をDBに保存し、長期記憶として管理する |
| FR-01-06 | RAGで過去会話を検索し、文脈に応じた返答を生成する |
| FR-01-07 | 記念日・過去の発言を参照した返答を生成できる（例：「昨日〜って言ってたよね」） |
| FR-01-08 | 開発時はGemini API、本番時はAmazon Bedrockに切り替え可能な設計にする |

### FR-02: Unit② — 自動注文モック（Auto-Order Mock）

| ID | 要件 |
|---|---|
| FR-02-01 | React製のUberEats風ダミーサイトと、Amazon風ダミーサイトを提供する |
| FR-02-02 | 「なんかご飯頼んで」などの発言でBotが意図を抽出する |
| FR-02-03 | PlaywrightがバックグラウンドサービスとしてダミーUIを自動操作する |
| FR-02-04 | 店舗検索→商品選択→カート投入→注文確認→注文完了のフローを自動実行する |
| FR-02-05 | Botが自動で注文を完了し、注文内容・金額・住所をメールで通知する。メールから5分以内のキャンセルが可能 |
| FR-02-06 | 確率的ミスロジック（大量注文ハプニング等）を実装し、Botが謝罪する |
| FR-02-07 | 1日の上限金額設定・5分間キャンセル機能を提供する |

### FR-03: Unit③ — 朝モーニングコール（Wake-up Routine）

| ID | 要件 |
|---|---|
| FR-03-01 | ユーザー設定時刻にBotが音声で話しかける（Webアプリ内音声自動再生） |
| FR-03-02 | Google Calendar APIと連携し、重要予定を検出する |
| FR-03-03 | 重要予定がある場合はミスパターンを抑制する（安全弁） |
| FR-03-04 | 確率的ミスパターンエンジンを実装する（通常65%・遅延20%・古い情報10%・スキップ5%） |
| FR-03-05 | 会話の中で睡眠時間が7時間取れるよう、就寝を勧める声かけを行う |
| FR-03-06 | 起床通知5分後に応答チェックを行い、未応答かつ重要予定がある場合は再通知する |
| FR-03-07 | パーソナライズされた起床メッセージをLLMで生成する |

### FR-04: Unit④ — 夜お姉さんタイム（Night Content）

| ID | 要件 |
|---|---|
| FR-04-01 | 毎晩22:00頃にBotがライフハック・トリビアを60〜90秒で紹介する |
| FR-04-02 | お姉さん風口調でコンテンツを提供する |
| FR-04-03 | 開発時は手動作成コンテンツDBを使用する |
| FR-04-04 | 本番時はユーザー会話ログからキーワード抽出し、LLMで事前生成してDBに保存するバッチ処理を実装する |

## 非機能要件

### NFR-01: パフォーマンス

| ID | 要件 |
|---|---|
| NFR-01-01 | 会話応答（LLM→TTS→再生開始）は5秒以内 |
| NFR-01-02 | VRMモデルのリップシンクは音声と50ms以内の同期 |
| NFR-01-03 | Playwright自動操作は30秒以内に完了する |

### NFR-02: セキュリティ（SECURITY拡張: 有効）

| ID | 要件 |
|---|---|
| NFR-02-01 | PostgreSQLは暗号化at-rest・TLS in-transitを有効化する（SECURITY-01） |
| NFR-02-02 | APIエンドポイントはすべて入力バリデーションを実施する（SECURITY-05） |
| NFR-02-03 | HTTPセキュリティヘッダー（CSP・HSTS・X-Content-Type-Options等）を設定する（SECURITY-04） |
| NFR-02-04 | ElevenLabs APIキー・Gemini APIキー・Bedrock認証情報はシークレットマネージャーで管理する（SECURITY-12） |
| NFR-02-05 | 構造化ログを実装し、PII・トークンをログに含めない（SECURITY-03） |
| NFR-02-06 | 依存パッケージはロックファイルで固定し、脆弱性スキャンをCIに組み込む（SECURITY-10） |
| NFR-02-07 | エラーレスポンスはユーザーに内部情報を露出しない（SECURITY-09・SECURITY-15） |
| NFR-02-08 | レート制限を公開APIエンドポイントに実装する（SECURITY-11） |

### NFR-03: スケーラビリティ・可用性

| ID | 要件 |
|---|---|
| NFR-03-01 | ハッカソンMVPはシングルユーザー想定（スケーリング不要） |
| NFR-03-02 | LLMプロバイダーの切り替えが設定変更のみで可能な設計にする |

### NFR-04: 保守性

| ID | 要件 |
|---|---|
| NFR-04-01 | Unit①〜④は独立したモジュールとして実装し、疎結合を保つ |
| NFR-04-02 | 環境変数で開発/本番の設定を切り替えられる |
| NFR-04-03 | バックエンド技術スタックは後から決定できるよう、インターフェース設計を先行する |

### NFR-05: テスト（PBT拡張: 部分適用）

| ID | 要件 |
|---|---|
| NFR-05-01 | 純粋関数（ミスパターンエンジン・記憶要約ロジック等）にプロパティベーステストを適用する（PBT-02・PBT-03） |
| NFR-05-02 | シリアライゼーション/デシリアライゼーションのラウンドトリップテストを実装する（PBT-02） |
| NFR-05-03 | PBTフレームワークを選定・設定する（PBT-09） |

## 倫理・安全設計要件

| ID | 要件 |
|---|---|
| ETH-01 | ダークパターン5条件を満たす設計にする（事前開示・0%設定可・自発的開示・事業者非利益・キャンセル可能） |
| ETH-02 | mistake_rateをユーザーが制御できる設定を提供する |
| ETH-03 | 連続ミス抑制ロジックを実装する |

## MVPスコープ（ハッカソン提出物）

**Inception（設計フェーズ）は全機能（Setup + Unit①〜④）を対象とする。**  
**実装フェーズ（Construction）のMVPは Setup + Unit①〜④ 全Unit。**

| 含む | 含まない |
|---|---|
| パートナー作成Setup | ユーザー認証 |
| 音声入力・会話 | |
| ElevenLabs音声合成 | |
| 方言対応 | |
| VRMモデル表示・リップシンク | |
| 長期記憶 + RAG | |
| Google Gemini（開発）/ Bedrock（本番）切り替え | |
| Unit②: UberEats風 + Amazon風ダミーサイト + Playwright自動注文 | |
| Unit②: メール通知 + 5分キャンセル | |
| Unit③: モーニングコール + Google Calendar連携 | |
| Unit③: 確率的ミスパターンエンジン | |
| Unit③: 睡眠7時間声かけ | |
| Unit④: 夜お姉さんタイム（22:00トリビア） | |
